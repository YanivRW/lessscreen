## What this app is

A native Android app that tracks how much time you spend in social media apps
(Instagram, TikTok, YouTube, Facebook, X, LinkedIn) and lets you compete with
friends on who uses them the **least**. Like a steps counter, but inverted —
lower minutes wins.

## How to get a new APK

Every push to `main` triggers `.github/workflows/build-apk.yml`. The workflow:
1. Restores the stable signing keystore from the GitHub secret `KEYSTORE_BASE64`
   (if the secret is missing, it generates a new one — **do not let this happen**
   or the SHA-1 will change and Google Sign-In will break again)
2. Builds a debug APK with `./gradlew :app:assembleDebug`
3. Uploads the APK as a GitHub Actions artifact

After a successful run, download the APK from:
- **Actions tab → latest run → Artifacts → LessScreen-debug-apk**
- Or use: `gh run download <run-id> -n LessScreen-debug-apk -D apk`

Releases are published manually with:
```bash
gh release create vX.Y.Z apk/app-debug.apk --title "..." --notes "..."
```

Latest release: https://github.com/YanivRW/lessscreen/releases

### Database tables (all in `public` schema)

| Table | Purpose |
|---|---|
| `profiles` | One row per user. Has `invite_code` (6-char, auto-generated by trigger on signup) |
| `friends` | Two rows per friendship (A→B and B→A). Mutual, auto-accept via invite code |
| `daily_usage` | One row per user per day. Stores `total_minutes` for that day |

### RLS
All three tables have Row Level Security enabled. Friends can read each other's
`daily_usage` rows. Profiles are readable by all authenticated users (needed for
invite code lookup).

### Auth trigger
A Postgres trigger `on_auth_user_created` fires after every new user signs up
and auto-creates a `profiles` row with a random 6-char `invite_code`.
Uses `md5(id || now())` — no `pgcrypto` extension needed.

### Email confirmation
**Currently: OFF** (disabled in Supabase Auth settings). Users sign up and
are immediately signed in. If you re-enable it, users will need to click an
email link before signing in.

## Google Sign-In configuration

### Google Cloud Console (project: LessScreen)
| Client | Type | Client ID |
|---|---|---|
| Android client 1 | Android | see Google Cloud Console → Credentials |
| LessScreen Web | Web application | see Google Cloud Console → Credentials |

- **Android client** registered with:
  - Package: `com.yanivrw.lessscreen`
  - SHA-1: see GitHub Actions build log → "Set up signing keystore" step → SHA1 line
- **Web client secret:** stored in Supabase → Authentication → Providers → Google (do not commit)
- **Test users added:** your Gmail (check OAuth consent screen → Test users)
- **OAuth consent screen:** External, Testing mode (not published)

### What's in the code
`GoogleConfig.WEB_CLIENT_ID` = the Web application client ID (not Android).
`AuthRepository.signInWithGoogle()` uses Android Credential Manager
(`GetSignInWithGoogleOption`) with a SHA-256 nonce, then exchanges the Google
ID token with Supabase via `supabase.auth.signInWith(IDToken)`.

### Google Sign-In
**Status:** Broken. Error: "No credentials available" from Android Credential Manager.

**What we tried:**
- `GetGoogleIdOption` with `filterByAuthorizedAccounts = false` → failed
- `GetSignInWithGoogleOption` → fails with same error
- Added SHA-256 nonce → no change
- Added test user in OAuth consent screen → no change
- All previous attempts used the **wrong client ID** (Android client instead of Web client)

**Current state:** v0.4.1 uses the correct Web client ID. This build has NOT
been confirmed working yet — it was just pushed and released before the user
had a chance to test.

**Most likely remaining issues to debug:**
1. Confirm Supabase Google provider has the Web Client ID + secret saved
2. Test v0.4.1 — if it still fails, the next thing to try is switching to
   the legacy `GoogleSignInClient` from `com.google.android.gms:play-services-auth:21.2.0`
   which is more reliable than Credential Manager for apps not on Play Store

**Workaround:** Email/password sign up/sign in works perfectly.

## Known quirks

- The scoreboard chart will only show one line (yours) until friends join and
  accumulate usage data. It needs at least one day of data per user to draw lines.
- `UsageStatsManager` only returns data for apps the user has actually opened.
  New installs or first-time permission grants may show 0 min for some apps
  until those apps are opened.
- Debug APK shows "Install from unknown source" warning on Android — expected,
  tap through to install.
- `KEYSTORE_BASE64` GitHub secret must stay set or the SHA-1 changes and breaks
  Google Sign-In registration.
